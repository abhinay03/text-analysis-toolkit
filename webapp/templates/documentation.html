Tokenizer

Core text preprocessing module. Splits text into tokens, handles punctuation removal, lowercasing, stopword filtering, and n-gram generation.

Constructor

Tokenizer(
    lower: bool = True,
    remove_punct: bool = True,
    remove_stopwords: bool = False,
    stopwords: Optional[List[str]] = None
)
lower Convert all text to lowercase before tokenization.
remove_punct Strip punctuation characters using regex [^\w\s].
remove_stopwords Filter out common English stopwords (90+ built-in words).
stopwords Custom stopword list. Falls back to defaults if None.

Methods

tokenize(text: str) -> List[str]

Splits text into word tokens. Applies lowercasing, punctuation removal, and stopword filtering based on configuration.

tk = Tokenizer(lower=True, remove_punct=True)
tk.tokenize("Hello, World! How are you?")
# ['hello', 'world', 'how', 'are', 'you']
sentence_split(text: str) -> List[str]

Splits text into sentences using . ! ? as delimiters.

tk.sentence_split("Hello world. How are you?")
# ['Hello world.', 'How are you?']
ngrams(tokens: List[str], n: int = 2) -> List[str]

Generates contiguous sequences of n tokens from the token list.

tk.ngrams(["hello", "world", "how", "are", "you"], 2)
# ['hello world', 'world how', 'how are', 'are you']

Stopwords

The built-in stopword list contains over 90 common English words including articles (a, an, the), prepositions (in, on, at, to), pronouns (i, you, he, she), and conjunctions (and, or, but). Access them via the read-only stopwords property.

FrequencyAnalyzer

Computes word frequency statistics from a list of tokens. Uses collections.Counter under the hood for efficient counting.

Properties

Total number of tokens (including duplicates).

Number of distinct tokens.

Methods

Returns a dictionary mapping each word to its count.

Returns the n most common words with their counts.

Term frequency (TF) of a word: count / total_words.

Relative frequency for every word in the corpus.

Filters words with count greater than the threshold.

SentimentAnalyzer

Lexicon-based sentiment analysis using a built-in dictionary of ~50 positive and ~50 negative words. Computes sentiment scores in the range [-1, 1] with configurable labels.

Constructor

SentimentAnalyzer(
    positive_words: Optional[Set[str]] = None,
    negative_words: Optional[Set[str]] = None
)

Score Calculation

The sentiment score is calculated as:

score = (positive_count - negative_count) / (positive_count + negative_count)

This produces a value between -1 (all negative) and +1 (all positive). A score of 0 means neutral or no sentiment words found.

Sentiment Labels

Score RangeLabel
> 0.3positive
< -0.3negative
-0.3 to 0.3neutral

Methods

score(tokens: List[str]) -> float

Returns a single sentiment score in [-1, 1].

detailed_score(tokens: List[str]) -> Dict

Returns a comprehensive breakdown including counts, ratios, score, and label.

{
    "positive_count": 4,
    "negative_count": 1,
    "positive_ratio": 0.25,
    "negative_ratio": 0.0625,
    "sentiment_score": 0.6,
    "sentiment_label": "positive"
}
positive_matches(tokens) -> List[str]

Returns all tokens that match the positive lexicon.

negative_matches(tokens) -> List[str]

Returns all tokens that match the negative lexicon.

load_custom_lexicon(filepath, sentiment) -> None

Loads additional words from a file (one word per line) into the positive or negative lexicon.

TextAnalyzer

The unified facade that coordinates the entire analysis pipeline. Combines tokenization, frequency analysis, sentiment scoring, and n-gram generation into a single call.

Constructor

TextAnalyzer(
    remove_stopwords: bool = False,
    custom_stopwords: Optional[List[str]] = None,
    ngram_range: Tuple[int, int] = (2, 3),
    report_title: str = "Text Analysis Report"
)

Methods

analyze(text: str) -> Dict

Runs the full analysis pipeline on a string. Returns a dictionary with keys: statistics, frequencies, sentiment, ngrams, tokens.

analyze_file(filepath: str, encoding="utf-8") -> Dict

Reads a file and runs the full analysis pipeline.

report_text(result: Dict) -> str

Convenience wrapper that generates a text report from an analysis result.

report_json(result: Dict) -> str

Convenience wrapper that generates a JSON report from an analysis result.

CLI Reference

Command-line interface for analyzing text files or inline text directly from the terminal.

text-analysis [file] [options]

Options

FlagDescription
filePath to text file (positional, optional if -t used)
-t, --textAnalyze inline text instead of a file
-o, --outputWrite report to file
-f, --formatOutput format: text (default) or json
--no-stopwordsRemove common stopwords
--ngram-minMinimum n-gram size (default: 2)
--ngram-maxMaximum n-gram size (default: 3)

Examples

# Analyze a file
text-analysis samples/positive_review.txt

# JSON output
text-analysis samples/mixed_article.txt -f json

# With stopword removal
text-analysis samples/negative_review.txt --no-stopwords

# Inline text
text-analysis -t "I love this amazing product!"

# Save to file
text-analysis samples/article.txt -o report.txt

# Custom n-gram range
text-analysis samples/article.txt --ngram-min 1 --ngram-max 4
